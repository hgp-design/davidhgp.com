# Audit — 2026-03-25

## Summary

Small, well-structured codebase. Token system is consistently applied, component boundaries are clean, and the pages/layouts/components split is working. Two real issues: a zombie CSS file that survived a refactor and tooltip styles living in the wrong place. The hand-crafted visual components (ToolChaosField, ToolPairingsGrid) have verbose CSS by nature — that's intentional and not a concern at this scale.

---

## Issues

### [High] `src/App.css` is a dead file

`App.css` (112 lines) is never imported anywhere — `App.jsx` has no CSS import. Its contents are a near-identical copy of `src/pages/Home.css` (which is the real file, imported by `Home.jsx`). `Home.css` has the 5 additional lines that `App.css` is missing. `App.css` is a zombie left over from before the pages refactor.

**Fix:** Delete `src/App.css`.

---

### [Medium] Tooltip styles are scoped to `Home.css` but used site-wide

`[data-tooltip]` and `[data-tooltip]::after` styles live in `src/pages/Home.css` (lines 58–83). Tooltips are a shared UI primitive — if they're ever used on a case study or future page, they won't render correctly unless that page also imports `Home.css`, which would be wrong.

**Fix:** Move the `[data-tooltip]` block to `src/index.css` alongside the other global tokens and base styles.

---

### [Low] `CLAUDE.md` has no project goals or context

`CLAUDE.md` is all technical rules — stack, file structure, hard constraints. There's no description of what the site is for, what impression it should make, or what "done" looks like. Claude Code gets the rules but not the intent, which matters when making judgment calls on copy, layout direction, or component scope.

**Fix:** Add a short "Purpose" or "Goals" section to `CLAUDE.md` describing the site's audience and goals.

---

## Clean passes

- Token compliance: all CSS uses `var(--*)` — no raw hex, rgb(), or hsl() outside `:root`
- Spacing: all spacing values use `--space-*` tokens or consistent 4px multiples
- No inline styles found in any `.jsx` file
- Component placement: all components are under `src/components/`, pages under `src/pages/`
- `App.jsx` is router-only — no logic or UI
- File sizes: only `BackgroundAnimation.jsx` (210 lines) and `ToolChaosField.css` (299 lines) exceed 150 lines; both are justified by their complexity
- `docs/decisions.md` is up to date with 11 logged decisions
